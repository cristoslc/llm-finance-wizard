# Documentation

Project documentation organized by artifact type. Each subdirectory contains its own README with details and a lifecycle index (`list-<type>.md`).

See [AGENTS.md](../AGENTS.md) for the documentation lifecycle workflow, artifact types, and relationship rules.

## Artifact Directories

| Directory | Artifact Type | Description |
|-----------|--------------|-------------|
| [vision/](vision/) | Product Vision | High-level product direction and strategy |
| [journey/](journey/) | User Journey | End-to-end user experience maps with Mermaid diagrams |

## Planned Directories

The following directories will be created as artifacts of these types are needed:

- `epic/` — Strategic initiatives and major capabilities
- `story/` — Atomic user-facing requirements
- `spec/` — Agent behavior contracts
- `research/` — Research spikes and investigations
- `adr/` — Architectural decision records
- `persona/` — User archetypes and research data
